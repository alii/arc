# M5 — `rt_js_ops`: full JS operator surface (threaded)

## Files
- **`2core/src/twocore/runtime/rt_js_ops.gleam`** — typed Gleam facade; every public fn is `@external(erlang, "twocore_rt_js_ops_ffi", ...)`. Mirrors the shape of existing `rt_js.gleam` (2core/src/twocore/runtime/rt_js.gleam:1-198).
- **`2core/src/twocore_rt_js_ops_ffi.erl`** — the implementation. Extends/replaces the arithmetic/compare surface currently in `twocore_rt_js_ffi.erl:60-353`.

## Dependencies
- **M1 `rt_js_store`**: for the `{js_cell, Int}` handle shape (matched in `js_type/1`) and no direct store access — M5 never dereferences a cell itself.
- **M3 `rt_js_val`** (the load-bearing dep): `t_to_primitive(St, V, Hint) -> {Prim, St'}`, `t_to_numeric(St, V) -> {Num|{js_bigint,N}, St'}`, `t_to_number(St, V) -> {Num, St'}`, `t_to_string(St, V) -> {Bin, St'}`, `is_bigint/1`, `is_object/1`, `js_type/1`, and the throw helpers `t_throw_type_error(St, Msg) -> no_return()` / `t_throw_range_error(St, Msg) -> no_return()` (which allocate an Error object in St then raise per M17's `{js_exn, St', Handle}` ABI). M5 owns NO coercion logic; it composes M3's.
- **M4 `rt_js_obj`**: `t_get_prop(St, Obj, Key) -> {V, St'}` (for `instanceof`'s `@@hasInstance` lookup + `.prototype` read) and `t_get_prototype_of(St, Obj) -> {Proto|null, St'}` (for the OrdinaryHasInstance chain walk).
- **M17 exception ABI**: consumed indirectly via M3's throw helpers.

## Full public signature list

### Arithmetic — threaded, `#(V, St)`
Reference semantics: `twocore_rt_js_ffi.erl:139-257` (numeric core, kept as private helpers `nadd/nsub/nmul/ndiv/nmod/nneg` — REUSED verbatim) + `arc/src/arc/vm/ops/numeric.gleam:134-224` (pow) + `arc/src/arc/vm/ops/operators.gleam:245-269` (BigInt arms).

```gleam
pub fn t_add(st: St, a: V, b: V) -> #(V, St)   // §13.15.3 ApplyStringOrNumericBinaryOperator "+"
pub fn t_sub(st: St, a: V, b: V) -> #(V, St)
pub fn t_mul(st: St, a: V, b: V) -> #(V, St)
pub fn t_div(st: St, a: V, b: V) -> #(V, St)
pub fn t_mod(st: St, a: V, b: V) -> #(V, St)
pub fn t_pow(st: St, a: V, b: V) -> #(V, St)   // §6.1.6.1.3 Number::exponentiate — NEW (not in current ffi)
```
**Algorithm (all six share this spine; `t_add` differs at step 2):**
1. `{Ap, St1} = rt_js_val:t_to_primitive(St, A, default)` (for `t_add`) or `number` hint (others). Source: arc interpreter.gleam:6063-6075 + 5972-5997.
2. `t_add` only: `{Bp, St2} = t_to_primitive(St1, B, default)`; if `is_binary(Ap) or is_binary(Bp)` → `{As, St3} = t_to_string(St2, Ap)`, `{Bs, St4} = t_to_string(St3, Bp)`, return `{<<As/binary, Bs/binary>>, St4}`. Else fall through with `Ap,Bp` as numeric operands.
3. `{An, St2} = rt_js_val:t_to_numeric(St1, Ap)`, then `{Bn, St3} = t_to_numeric(St2, Bp)`.
4. Dispatch on `{An, Bn}` shape:
   - `{{js_bigint,X}, {js_bigint,Y}}` → BigInt path (below).
   - `{{js_bigint,_}, _}` or `{_, {js_bigint,_}}` → `rt_js_val:t_throw_type_error(St3, <<"Cannot mix BigInt and other types, use explicit conversions">>)` (arc operators.gleam:227).
   - `{Num, Num}` → apply the sentinel-aware kernel (`nadd/nsub/nmul/ndiv/nmod/npow`) from twocore_rt_js_ffi.erl:158-257, return `{out(Result), St3}`.
5. BigInt path per op (arc operators.gleam:245-298): `sub→X-Y`, `mul→X*Y`, `div→ if Y==0 then t_throw_range_error(St3,<<"Division by zero">>) else X div Y`, `mod→` same guard `else X rem Y`, `pow→ if Y<0 then t_throw_range_error else bigint_pow(X,Y,1)` (squaring, operators.gleam:273-281), `add→X+Y`. Return `{{js_bigint, R}, St3}`.

`npow/2` (new private helper, port of arc numeric.gleam:134-278): full §6.1.6.1.3 case ladder — NaN exp → nan; exp==±0 → 1; NaN base → nan; ±Inf base/exp table; ±0 base with odd-integer test; finite×finite via `math:pow/2` wrapped in `try…catch error:badarith → signed_inf(...)` for overflow, plus the `X<0 andalso not is_integral(Y) → nan` guard.

### Bitwise — threaded, `#(V, St)` where V is a BEAM integer
Source: arc operators.gleam:106-138, 284-298, 382-409; arc numeric.gleam:295-329.
```gleam
pub fn t_bitand(st: St, a: V, b: V) -> #(V, St)
pub fn t_bitor (st: St, a: V, b: V) -> #(V, St)
pub fn t_bitxor(st: St, a: V, b: V) -> #(V, St)
pub fn t_shl   (st: St, a: V, b: V) -> #(V, St)
pub fn t_shr   (st: St, a: V, b: V) -> #(V, St)   // signed (arithmetic) >>
pub fn t_ushr  (st: St, a: V, b: V) -> #(V, St)   // >>> — the ONE op returning [0, 2^32)
```
**Algorithm:** `{An, Bn, St3} = to_numeric_pair(St, A, B)` (steps 1+3+4-mix-check above, `number` hint). Then:
- BigInt×BigInt: `and→X band Y`, `or→X bor Y`, `xor→X bxor Y`, `shl→X bsl Y`, `shr→X bsr Y` (Erlang bsl/bsr accept negative counts = shift other way, matching §6.1.6.2.9/10 — arc operators.gleam:291-294), `ushr→ t_throw_type_error(St3, <<"BigInts have no unsigned right shift, use >> instead">>)`. Return `{{js_bigint, R}, St3}`.
- Number×Number: `A32 = to_int32(An)`, `B32 = to_int32(Bn)` (ushr: `to_uint32`). Raw: `and→A32 band B32`, `or→bor`, `xor→bxor`, `shl→A32 bsl (B32 band 31)`, `shr→A32 bsr (B32 band 31)`, `ushr→to_uint32(An) bsr (B32 band 31)`. Then `wrap_int32(Raw)` (ushr: `wrap_uint32(Raw)`). Return `{Wrapped, St3}`.

**Private helpers** (port arc numeric.gleam:295-329 to Erlang):
```erlang
to_int32(N)    -> case N of nan->0; inf->0; neg_inf->0; F when is_float(F) -> wrap_int32(trunc(F)); I -> wrap_int32(I) end.
to_uint32(N)   -> case N of nan->0; inf->0; neg_inf->0; F when is_float(F) -> wrap_uint32(trunc(F)); I -> wrap_uint32(I) end.
wrap_uint32(I) -> I band 16#FFFFFFFF.
wrap_int32(I)  -> U = I band 16#FFFFFFFF, case U > 16#7FFFFFFF of true -> U - 16#100000000; false -> U end.
```

### Relational — threaded, `#(Int, St)` (i32 1|0)
Source: twocore_rt_js_ffi.erl:265-312 (kernel `ncmp` reused) + arc operators.gleam:455-560 (BigInt vs Number/String compare).
```gleam
pub fn t_lt(st: St, a: V, b: V) -> #(Int, St)
pub fn t_le(st: St, a: V, b: V) -> #(Int, St)
pub fn t_gt(st: St, a: V, b: V) -> #(Int, St)
pub fn t_ge(st: St, a: V, b: V) -> #(Int, St)
```
**Algorithm — full §7.2.13 Abstract Relational Comparison:**
1. `{Ap, St1}=t_to_primitive(St,A,number)`, `{Bp, St2}=t_to_primitive(St1,B,number)` (LeftFirst — for `>`/`>=` the emitter still evaluates operands L→R and calls `t_gt` — no swap needed since ToPrimitive order is fixed by evaluation order, and `t_gt` internally computes `cmp(Ap,Bp)` then tests `gt`).
2. If both binaries → **byte-wise binary compare** (D10-1: Erlang `Ap < Bp` on binaries; UTF-8 byte order, NOT UTF-16 code-unit order — deliberate divergence from §7.2.13 step 3, since JS strings are stored UTF-8 in this runtime).
3. If `{js_bigint,X}` vs binary → StringToBigInt(Bp): `try binary_to_integer(string:trim(Bp))` else `undefined` (NaN-ish). Compare as BigInt or return `undefined`.
4. `{An, St3}=t_to_numeric(St2,Ap)`, `{Bn, St4}=t_to_numeric(St3,Bp)`. BigInt×BigInt → integer compare. BigInt×Number → arc operators.gleam:528-554 (`compare_bigint_num`: nan→undefined; ±inf→lt/gt; finite→compare `X` vs `trunc(F)` with fractional tiebreak). Number×Number → `ncmp` (twocore_rt_js_ffi.erl:299-312).
5. Map `lt/eq/gt/undefined` to 1|0 per op (`undefined` → always 0). Return `{Int, St4}`.

### instanceof — threaded, `#(Int, St)`
Source: arc/src/arc/vm/ops/instanceof.gleam:28-210 (full port).
```gleam
pub fn t_instanceof(st: St, v: V, target: V) -> #(Int, St)
```
**Algorithm (§13.10.2 InstanceofOperator):**
1. If `target` is not `{js_cell,_}` and not `fun()` → `t_throw_type_error(St, <<"Right-hand side of 'instanceof' is not an object">>)`.
2. `{Handler, St1} = rt_js_obj:t_get_method(St, Target, {js_sym, has_instance})` (M4; GetMethod: undefined/null → absent).
3. If Handler present: `{R, St2} = call_closure(St1, Handler, Target, [V])` (uniform `f(State,This,Args)->{Res,State'}` ABI), return `{rt_js_val:truthy(R), St2}`.
4. Else if Target not callable (`not is_function(Target)` and not a callable cell per M4's `is_callable`) → `t_throw_type_error(St1, <<"Right-hand side of 'instanceof' is not callable">>)`.
5. Else `t_ordinary_has_instance(St1, Target, V)`.

```gleam
pub fn t_ordinary_has_instance(st: St, ctor: V, v: V) -> #(Int, St)  // exported — Function.prototype[@@hasInstance] and Iterator.from need it (arc iterator.gleam:269)
```
Body: bound-function unwrap (M4 exposes `[[BoundTargetFunction]]` slot) → recurse `t_instanceof`; else if `V` not object → `{0, St}`; else `{P, St1}=rt_js_obj:t_get_prop(St,Ctor,<<"prototype">>)`; if `P` not object → throw TypeError; else walk: `t_get_prototype_of` loop comparing `{js_cell,N}=:=P` by handle identity, fuel-bounded at 10000 (arc limits.max_prototype_depth) → RangeError on exhaustion.

## Invariants M5 upholds
1. **Threading**: every `t_*` function returns `{Result, St'}` where `St'` reflects every store mutation performed by ToPrimitive/ToNumeric callbacks (user `valueOf`/`toString`/`@@toPrimitive` may allocate). On throw, the `St` passed to `rt_js_val:t_throw_*` is the fully-updated store at the throw point.
2. **Bitop range**: `t_bitand/bitor/bitxor/shl/shr/bitnot` on Number operands return a BEAM integer in `[-2^31, 2^31)`. `t_ushr` on Number operands returns a BEAM integer in `[0, 2^32)`. On BigInt operands, return `{js_bigint, N}` (unbounded). (arc operators.gleam:388, numeric.gleam:315-329.)
3. **Left-to-right coercion**: `A` is coerced before `B` in every binary op (observable via side-effecting `valueOf`). `t_gt/t_ge` do NOT swap operands — they compute the same L-then-R comparison and test the `gt`/`ge` outcome.
4. **No direct store access**: M5 never pattern-matches inside a `{js_cell,_}` payload. All object introspection goes through M4.
5. **BigInt/Number mixing**: any arithmetic/bitwise/`t_plus` mixing → TypeError (msg from arc operators.gleam:227). Relational + loose-eq DO allow mixing (§7.2.13/§7.2.14).
6. **NaN propagation**: any arithmetic with a `js_nan` operand (post-coercion) yields `js_nan`. Any relational with NaN yields `0`. `strict_eq(nan,nan)=0`, `t_eq(nan,nan)=0`.

## Fast-path IR pattern (what M12 `emit_2core/expr` emits per op)
The IR nodes: `TermTest(IsNumber, v)` → i32 1|0 (ir.gleam:939, 1512); `NumTerm(op, a, b)` (ir.gleam:965, 1530-1539: `NAdd NSub NMul NLt NLe NGt NGe NEq` only — no div/mod/pow, no bitwise); `CallHost("js", op, [args])` (ir.gleam:665).

| JS op | IR emitted (nested-If since IR has no `&&`) |
|---|---|
| `a + b` | `If(TermTest(IsNumber,a), If(TermTest(IsNumber,b), NumTerm(NAdd,a,b), SLOW), SLOW)` where `SLOW = CallHost("js","add",[a,b])` |
| `a - b`, `a * b` | same shape with `NSub`/`NMul` and `"sub"`/`"mul"` |
| `a / b`, `a % b`, `a ** b` | **always** `CallHost("js","div"/"mod"/"pow",[a,b])` — no NumTerm (BEAM `/0` traps, `rem 0` traps, no `**` BIF). HANDOFF §5 line 170. |
| `a < b`, `<=`, `>`, `>=` | `If(TermTest(IsNumber,a), If(TermTest(IsNumber,b), NumTerm(NLt/NLe/NGt/NGe,a,b), SLOW), SLOW)` |
| `a === b` | `If(TermTest(IsNumber,a), If(TermTest(IsNumber,b), NumTerm(NEq,a,b), SLOW), SLOW)`; `!==` wraps in `1 - result` |
| `a == b`, `!=` | **always** `CallHost("js","eq"/"neq",[a,b])` (fast-path `NEq` unsafe: `1 == "1"` must be true) |
| `a & \| ^ << >> >>>` b | **always** `CallHost("js","bitand"/…,[a,b])` (no scalar bitwise IR node — ir.gleam grep confirms only v128 bitwise at :1183) |
| `-a`, `+a`, `~a` | **always** `CallHost("js","neg"/"plus"/"bitnot",[a])` (no unary NumTerm) |
| `typeof a` | `CallHost("js","type_of",[a])` — pure dispatch arm |
| `a instanceof b` | `CallHost("js","instanceof",[a,b])` |

**Gotcha**: the fast path fires only when BOTH operands are BEAM `is_number/1` — sentinel atoms (`js_nan/js_inf/js_neg_inf`), BigInts, and strings all fail the guard and hit SLOW. So `Infinity + 1` correctly goes slow-path. `NAdd` on two integers yields a bignum (never overflows, never traps) — the documented int-as-smi divergence (twocore_rt_js_ffi.erl:36-38) is inherent; M3 owns whether/how to normalize.

## Gotchas / edge cases the implementer must not miss
- **`strict_eq` is JPure** — no `St` param, no tuple return. Emitter treats it exactly like `type_of`. Do NOT wrap in a threaded shim.
- **Shift count masking**: Number shifts mask count to 5 bits (`band 31`) BEFORE shifting (arc operators.gleam:125,129,135). BigInt shifts do NOT mask — Erlang `bsl`/`bsr` accept any integer count including negative (arc operators.gleam:291-294).
- **`t_plus` on BigInt throws** (§13.5.6 / arc operators.gleam:205-209) — unary `+` is the one place BigInt is rejected pre-ToNumeric.
- **`npow` overflow**: `math:pow/2` raises `badarith` on overflow (e.g. `math:pow(1.0e308, 2)`) — must `try…catch → signed_inf(...)`.
- **`bnot` needs no post-wrap** on Number path (result of `bnot(X)` for X∈[-2^31,2^31) stays in range: `bnot(X) = -X-1`).
- **`instanceof` closure call**: uses the uniform compiled-fn ABI `Fun(St, This, Args) -> {R, St'}` where `Args` is a BEAM list. So `Handler(St1, Target, [V])`.
- **Prototype-walk fuel**: hard-code `10000` (or import `rt_js_obj`'s constant) — a `getPrototypeOf` proxy trap returning fresh proxies would otherwise loop forever (arc instanceof.gleam:178-185).

## Build order
M5 can be **written in parallel with M1/M4** (interface-only dependency), but **compiles only after M3** (`rt_js_val` must export `t_to_primitive/3`, `t_to_numeric/2`, `t_to_string/2`, `t_throw_type_error/2`, `t_throw_range_error/2`, `truthy/1`, `js_type/1`). The numeric kernels (`nadd`…`nmod`, `ncmp`, `to_int32`…) have zero deps and can be written+unit-tested day one by lifting from `twocore_rt_js_ffi.erl` + `arc/src/arc/vm/ops/numeric.gleam`.
