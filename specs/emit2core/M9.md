## §0. LOAD-BEARING CORRECTION to a "fixed" decision

The task's fixed decision says `f(State, This, Args) -> {State', Result}` (state-first return tuple). **2core's existing Threaded plumbing universally uses `{Result, St'}` — VALUE FIRST, STATE SECOND:**
- `emit_return` (emit_core.gleam:1556-1560): `CTuple([pkg, CVar(cur)])`
- `emit_threaded_call_unpack` (:3490-3513): matches `PTuple([PVar(pkgvar), PVar(stvar)])`
- `emit_value_state_pair` (:1919-1940): matches `PTuple([PVar(vvar), PVar(stvar)])`
- `emit_exports` pure-adapter (:436): `{apply 'fn'/n(A…), St}`
- `apply_cont` KReturn/Threading (:1998+ via emit_return): pairs `{Package, cur}`
- `rt_state.t_grow` etc. return `#(Int, InstanceState)`

Flipping to `{St', V}` would require rewriting ~8 helpers + every `t_*` in rt_state/rt_mem/rt_table. **VERDICT: spec `{Result, St'}` (value first).** All rt_js threaded ops return `#(JsVal, InstanceState)` (or `#(a, InstanceState)` for typed results) and reuse `emit_value_state_pair`/`emit_threaded_call_unpack` verbatim.

**Compiled JS fn signature (final):** IR-level `f(cap₁,…,capₘ, _frame, _args)` (m+2 params), where `_frame` is the M-CALL `Frame` 4-tuple `{this, active_func, home_object, new_target}`. Under Threaded it emits at `'f'/(m+3) = fun(St, cap₁,…,capₘ, Frame, Args) -> {Result, St'}`. Closure runtime shape: `fun/3 = fun(St, Frame, Args) -> {V, St'}`.

## §3. New types in emit_core.gleam (insert after `js_capability` at :3602)

```gleam
/// Which split JS-runtime module a `CallHost("js", op, …)` targets under `js_profile`.
/// D3a: a CLOSED build-time enum → resolved to a Binding field atom by `js_module_atom`;
/// the target module is NEVER derived from the `op` string at runtime.
type JsRtModule {
  JsStore   // M1 → binding.js_store_module
  JsVal     // M3 → binding.js_val_module
  JsObj     // M4 → binding.js_obj_module
  JsOps     // M5 → binding.js_ops_module
  JsClass   // M7 → binding.js_class_module
  JsAsync   // M8 → binding.js_async_module
}

/// How a `rt_js` op interacts with the threaded `InstanceState`.
type JsOpKind {
  /// No `St` param, single-value return. `cur` flows through unchanged.
  /// Emit: `call 'mod':'fn'(args…)` → `apply_cont_call(cont, call, 1, sc, …)`.
  JPure
  /// Leading `St` param, single-value return (reads store, never mutates — no GC).
  /// `sc` is UNCHANGED (cur flows through). Emit: `call 'mod':'t_fn'(cur, args…)`
  /// → `apply_cont_call(cont, call, 1, sc, …)` with NO state-pair unpack.
  JRead
  /// Leading `St` param, `{V, St'}` return (may allocate/mutate/GC/call user code).
  /// Emit: `call 'mod':'t_fn'(cur, args…)` → `emit_value_state_pair(call, cont, …)`.
  JMut
  /// Leading `St` param, bare `St'` return (no value — effect-only: cell_set,
  /// pin_root, drain_microtasks). Rebinds `sc` to `St'`; result is discarded.
  /// Emit: NOT via `emit_threaded_record_effect` (that Result-wraps) — instead a
  /// direct `Let([st2], call, <cont under Threading(st2) with []>)`.
  JMutUnit
}

/// One resolved JS-runtime op: target module + Erlang function name + threading kind.
/// `arity` is the count of `args` the frontend passes (excluding the `St` param that
/// `JRead`/`JMut`/`JMutUnit` prepend); recorded for `ArityMismatch` checking.
type JsOp {
  JsOp(module: JsRtModule, fn_name: String, arity: Int, kind: JsOpKind)
}

fn js_module_atom(ctx: Ctx, m: JsRtModule) -> String {
  case m {
    JsStore -> ctx.binding.js_store_module
    JsVal -> ctx.binding.js_val_module
    JsObj -> ctx.binding.js_obj_module
    JsOps -> ctx.binding.js_ops_module
    JsClass -> ctx.binding.js_class_module
    JsAsync -> ctx.binding.js_async_module
  }
}
```

## §5. `emit_call_host` "js" branch rewrite — emit_core.gleam:3549-3566

**Current** (:3551-3566): `resolve_js(name) → Some(fn_name) → apply_cont_call(cont, CCall(js_runtime_module, fn_name, cargs), 1, sc, …)` — always state-neutral.

**New:**
```gleam
True ->  // capability == "js"
  case resolve_js(name) {
    None -> Error(UnknownJsOp(name))
    Some(JsOp(jmod, fn_name, expected_arity, kind)) -> {
      case list.length(args) == expected_arity {
        False -> Error(ArityMismatch(expected_arity, list.length(args)))
        True ->
          case ctx.binding.js_profile, sc, kind {
            // ── js_profile: False → legacy Phase-8 path (byte-identical) ──
            False, _, _ ->
              apply_cont_call(
                cont,
                CCall(CAtom(ctx.binding.js_runtime_module), CAtom(fn_name), cargs),
                1, sc, state, ctx,
              )
            // ── js_profile: True — dispatch by kind ──
            True, _, JPure ->
              apply_cont_call(
                cont,
                CCall(CAtom(js_module_atom(ctx, jmod)), CAtom(fn_name), cargs),
                1, sc, state, ctx,
              )
            True, Threading(cur), JRead ->
              // St prepended; result is bare V; sc UNCHANGED (no state-pair unpack).
              apply_cont_call(
                cont,
                CCall(CAtom(js_module_atom(ctx, jmod)), CAtom(fn_name),
                      [CVar(cur), ..cargs]),
                1, sc, state, ctx,  // cur unchanged
              )
            True, Threading(cur), JMut ->
              emit_value_state_pair(  // reuses helper at :1919-1940
                CCall(CAtom(js_module_atom(ctx, jmod)), CAtom(fn_name),
                      [CVar(cur), ..cargs]),
                cont, state, ctx,
              )
            True, Threading(cur), JMutUnit -> {
              // Bare St' return, no value. Rebind sc directly — NOT via
              // emit_threaded_record_effect (that Result-wraps for WASM effects).
              let #(st2, state2) = fresh_var(state)
              let call = CCall(CAtom(js_module_atom(ctx, jmod)), CAtom(fn_name),
                               [CVar(cur), ..cargs])
              use #(body, state3) <- result.map(
                apply_cont(cont, [], Threading(st2), state2, ctx))
              #(CLet([st2], call, body), state3)
            }
            // js_profile:True but sc=NoState is unreachable (universal threading, §8),
            // but fail closed with a typed error rather than crash:
            True, NoState, _ -> Error(JsProfileRequiresThreaded(name))
          }
      }
    }
  }
```

Add `JsProfileRequiresThreaded(op: String)` to `EmitError` (:194 area). `emit_value_state_pair` (:1919) is reused unchanged — it emits `case <call> of <{V, St2}> when 'true' -> <cont with [V] under Threading(St2)>`.

## §6. `emit_make_closure` rewrite — emit_core.gleam:1493-1522

**Current** (:1490-1492 doc): "state-threaded build whose closure target is state-reaching is out of scope". Remove this limitation.

**New logic** (inserted into the `True ->` arm at :1508 after the arity check):
```gleam
True -> {
  let m = list.length(captures)
  let target_threaded =
    is_threaded(ctx) && set.contains(ctx.fn_state_reaching, fn_name)
  case target_threaded {
    False -> {
      // UNCHANGED Phase-8 shape: fun(A₁…A_arity) -> apply 'f'/(m+arity)(caps, A…)
      let #(param_names, state2) = fresh_n_vars(state, arity)
      let body = CApply(FName(fn_name, m + arity),
        list.append(list.map(captures, emit_value),
                    list.map(param_names, CVar)))
      apply_cont(cont, [CFun(param_names, body)], sc, state2, ctx)
    }
    True -> {
      // NEW threaded shape: fun(St, A₁…A_arity) -> apply 'f'/(m+arity+1)(St, caps, A…)
      // The closure has runtime arity (arity+1); its body is a THREADED tail call, so
      // it returns the target's {Result, St'} straight through (§B.3 tail-call rule).
      let #(st_param, state1) = fresh_var(state)
      let #(param_names, state2) = fresh_n_vars(state1, arity)
      let body = CApply(FName(fn_name, m + arity + 1),
        [CVar(st_param),
         ..list.append(list.map(captures, emit_value),
                       list.map(param_names, CVar))])
      apply_cont(cont, [CFun([st_param, ..param_names], body)], sc, state2, ctx)
    }
  }
}
```

**Invariants:**
- Building the fun is still PURE (`sc` flows through `apply_cont` unchanged — line :1518 pattern preserved).
- The IR-level arity check (`fn_params == m + arity` at :1505-1507) is UNCHANGED — it checks against the IR `Function.params` count, which is m+2 for a JS fn (`captures + [this, args]`). `emit_function` (:641-652) independently emits at +1.
- Captured cell handles (`{js_cell, Int}`) are ordinary `TTerm` values in `captures` — no special handling.

**IR-level convention (M14 emits):** `MakeClosure(fn_name, captures, arity=2)` — arity is ALWAYS `2` (`[_frame, _args]`). emit_core adds the St slot → BEAM fun/3.

## §7. `emit_call_closure` rewrite — emit_core.gleam:1536-1545

**Current** (:1544-1545): `CApplyExpr(callee, args)` → `apply_cont_call(cont, applied, 1, sc, …)` — state-neutral (doc :1530-1533 says "cur flows through unchanged").

**New:**
```gleam
fn emit_call_closure(
  callee: Value, args: List(Value), cont: Cont, sc: StateChan,
  state: EmitState, ctx: Ctx,
) -> Result(#(CExpr, EmitState), EmitError) {
  let cargs = list.map(args, emit_value)
  case ctx.binding.js_profile, sc {
    // Legacy / non-JS: state-neutral, byte-identical.
    False, _ | True, NoState ->
      apply_cont_call(cont, CApplyExpr(emit_value(callee), cargs), 1, sc, state, ctx)
    // JS profile + Threaded: closure is a fun/3 returning {V, St'}.
    // Mirrors emit_call_direct's threaded arm (:3466-3471).
    True, Threading(cur) -> {
      let applied = CApplyExpr(emit_value(callee), [CVar(cur), ..cargs])
      case cont {
        // Tail position: {V, St'} is exactly what caller returns → straight through (§B.3).
        KReturn -> Ok(#(applied, state))
        // Non-tail: destructure {V, St'}, continue under Threading(St').
        _ -> emit_threaded_call_unpack(applied, 1, cont, state, ctx)
      }
    }
  }
}
```

`emit_threaded_call_unpack` (:3490-3513) reused verbatim: `case <applied> of <{Pkg, St'}> -> <apply_cont_call(cont, Pkg, 1, Threading(St'), …)>`.

**IR-level convention (M12 emits):** `CallClosure(callee, [frame_val, args_cons])` — always exactly 2 IR args. `frame_val` is the M-CALL `Frame` 4-tuple `{this, active_func, home_object, new_target}` (built via `TermOp(MakeTuple, …)`; see u-call-abi §Frame). emit_core prepends `cur` → BEAM `apply Callee(St, Frame, Args)`.

```gleam
  // JS profile: EVERY function threads St (they all reach JsStore via CallHost/
  // CallClosure). Short-circuit — no fixpoint needed.
  True -> set.from_list(list.map(module.functions, fn(f) { f.name }))
  False -> state_reaching_closure(module.functions)
}
```

**Rationale:** every JS function body contains at least one `CallHost("js",…)` (arg unpacking prologue does `t_cell_get`/`undefined_sentinel`, or the return path wraps in `t_to_object`) OR a `CallClosure` (any call expr). Rather than teach `expr_touches_state` about JMut ops + CallClosure + MakeClosure edges, force universal threading — simpler, correct, and matches the design intent (JsStore is the ONLY state a JS module has). WASM modules (`js_profile: False`) are byte-identical.

**Downstream effect:** `emit_function` (:641) sees every fn as state-reaching → all emit at `'f'/(n+1) = fun(St, params…) -> {Pkg, St'}`. `emit_exports` (:423-438) sees every export as state-reaching → all export at arity+1 with `{Pkg, St'}` shape (uniform run-ABI, unchanged logic).

## §9. Exception ABI seam (cross-refs M17)

**Current blocker** (emit_core.gleam:5127-5128): `emit_try` doc says "Cell-only (T6): the same `sc` flows to the handler (no state travels in the throw — Threaded+EH is a categorised-unsupported combo)." Confirmed in code: `emit_try` (:5145) emits body under `sc`, handler under **same** `sc` (:5168) — a body that mutates `St` then throws leaves the handler with the PRE-BODY `cur`. **This is wrong for JS:** `try { obj.x=1; throw 0 } catch { obj.x }` must see `1`.

**Also:** `emit_throw` (:5097-5106) doesn't take `sc` at all (call site :1147 passes `state, ctx` only) — it can't reach `cur`.

**M9 spec for the seam (M17 implements the emitter side):**
- JS uses ONE tag: every JS module's `Module.tags = [TagDecl("js_exn", [TTerm])]` — **IR-level** payload arity is **1** (`[v]`); `emit_core` under `Threading(cur)` prepends `St` to reach the **wire** payload `{wasm_exn, 0, [St, V]}` (D6). tag_index (:249, :701) resolves `"js_exn"` → `0`.
- `emit_throw` gains `sc: StateChan` param. Under `Threading(cur) && js_profile`, emit `call 'rt_exn':'throw_exn'(0, [cur, exc_val])` — `cur` prepended to payload.
- `emit_try` under `js_profile && Threading(_)`: `try_dispatch` (:5192) for `OnTag("js_exn")` binds payload to `[St', ExcVal]` and emits the handler body under `Threading(St')` (NOT the outer `sc`). The reraise default (:5203-5212) is unchanged — `rt_exn:reraise` propagates `{wasm_exn, 0, [St', ExcVal]}`; an outer JS catch re-destructures `St'` from the same payload.
- **rt_js ops that raise** (`t_not_callable`, `t_throw_type_error`, getter throws inside `t_get_prop`): raise via `erlang:throw({wasm_exn, 0, [St_at_throw, ErrObj]})` — the `St'` in the payload IS the state at the throw point (rt_js has it in hand).

**M17 owns:** the `finally` barrier-duplication rewrite + emit_throw/emit_try modifications. **M9 owns:** documenting that `js_profile` LIFTS the "Threaded+EH unsupported" restriction via the `[St', ExcVal]` payload convention, and adding `sc` to `emit_throw`'s signature.

## §10. Erlang-side `-spec` for the four op kinds (rt_js_*.erl reference)

```erlang
%% JPure — e.g. rt_js_ops:strict_eq/2
-spec strict_eq(term(), term()) -> 0 | 1.

%% JRead — e.g. rt_js_store:t_cell_get/2
-spec t_cell_get(instance_state(), {js_cell, integer()}) -> term().

%% JMut — e.g. rt_js_store:t_cell_new/2
-spec t_cell_new(instance_state(), term()) -> {{js_cell, integer()}, instance_state()}.
%% GENERAL: -spec t_<op>(instance_state(), Arg1, …, ArgN) -> {term(), instance_state()}.

%% JMutUnit — e.g. rt_js_store:t_cell_set/3
-spec t_cell_set(instance_state(), {js_cell, integer()}, term()) -> instance_state().
%% GENERAL: -spec t_<op>(instance_state(), Arg1, …, ArgN) -> instance_state().
```

Where `instance_state()` = the Gleam-record tuple `{instance_state, Mems, Globals, Tables, DD, DE, RG, FI, JsStore}` (9-tuple with new `js_store` at position 9).
