## §2 Native-function representation (DECISION for task item 4)

**CHOSEN: `NativeToken` sum-type dispatch, NOT a raw BEAM fun / NOT an `Int` id / NOT `{js_native, atom, arity}`.**

Rationale:
1. Storing `fn(St, ...) -> #(..., St)` in a store cell makes `Slot` recursively contain `St` → Gleam type cycle. arc dodges this via `Heap(ctx, host)` genericity + `Host(fn(...))` (value.gleam:2067) but that machinery isn't worth replicating when there's exactly one `St` type here.
2. `{js_native, ModAtom, FnAtom}` + `erlang:apply/3` violates D3a's spirit (2core emit_core.gleam:3604-3620: "target is NEVER constructed from program/runtime data"). Token→case-arm dispatch keeps the closed-set property.
3. Enum tokens are GC-traceable (variants can carry captured `Handle`s — needed for `PromiseAllResolveElement`, `BoundFunction`, `ErrorConstructor(proto)`) and pattern-matchable (needed for `Function.prototype.toString`, `typeof`, `instanceof`).
4. Direct precedent: arc value.gleam:2072 `NativeFn` + per-module `dispatch`.

**The callable slot kind** (lives in M1's `JsSlot` / `ObjKind` type, but the payload types live here in `common.gleam`):
```gleam
// in rt_js_store.JsSlot's ObjKind variant set (M1 owns the outer type):

KFunction(closure: CompiledFn, home: Option(Handle), this_mode: ThisMode, constructible: Bool)
   // `closure` is the BEAM fun/3 from MakeClosure — opaque `CompiledFn` (D16);
   // called via rt_js_builtins_ffi:apply_closure(F, St, This, Args) -> {Result, St'}
KBound(target: Handle, bound_this: JsVal, bound_args: List(JsVal))

pub type ThisMode { Lexical  Strict  Global }   // arrow vs strict-fn vs sloppy-fn
```

## §3 t_call dispatch (interface with M4)

`t_call` lives in **M4 rt_js_obj**, not M6, but M6 defines the `dispatch_native` it delegates to. Contract:

```gleam
// M4 rt_js_obj.gleam — spec'd here for the seam
pub fn t_call(st: St, callee: JsVal, this: JsVal, args: List(JsVal), new_target: Option(JsVal)) -> #(JsResult, St)
```
Algorithm:
1. If `callee` is a `Handle` → `rt_js_store.t_get(st.store, h)` → match `slot.kind`:
   - `KNative(tok, _)` → `rt_js_builtins.dispatch_native(st, tok, this, args, new_target)`
   - `KFunction(f, _, mode, _)` → resolve `this` per `mode` (Global→`st.realm.global_this`, Lexical→passed as-is, Strict→as-is), then `rt_js_builtins_ffi:apply_closure(f, st, resolved_this, args)` returning `#(JsResult, St)` directly
   - `KBound(tgt, bt, ba)` → `t_call(st, {js_cell,tgt}, bt, list.append(ba, args), new_target)`
   - `ProxyObject(Some(slots), callable=True, ..)` → proxy [[Call]] trap
   - else → `#(Error(type_error(st, "not a function")), st)`
2. If `callee` is a raw BEAM fun (fast path — emitter emits `CallClosure` directly when it statically knows callee is a KFunction): unreachable from `t_call` — `t_call` is only reached via `CallHost("js","call",...)` for the DYNAMIC case.
3. Else → TypeError.

```erlang
%% rt_js_builtins_ffi.erl
-spec apply_closure(fun(), St, term(), list()) -> {{ok,term()} | {error,term()}, St}.
apply_closure(F, St, This, Args) -> F(St, This, Args).
```

## §4 Uniform builtin signature & JsResult

```gleam
pub type JsResult = Result(JsVal, JsVal)   // Ok(v) | Error(thrown_value)
// Gleam→Erlang wire: {ok, V} | {error, E}
```
**Every builtin impl**: `fn(st: St, this: JsVal, args: List(JsVal), nt: Option(JsVal)) -> #(JsResult, St)` — where `St` = the full threaded state (rt_js_state.St, which carries `store: JsStore`, `realm: Realm`). `JsStore` in turn carries `host_hooks: HostHooks` (monotonic_now, random, print, sleep_ms — see SPEC.md §2.4) — builtins that observe the host (Date, Math.random, console, performance) read through `st.store.host_hooks`, NEVER via direct Erlang calls.

`nt` (new_target) is `Some(ctor)` when reached via `[[Construct]]`, `None` for `[[Call]]`. Constructors that require `new` check `nt` and throw TypeError on `None`; `Symbol()`/`BigInt()` throw on `Some`.

**INVARIANT: builtins NEVER `erlang:error`/`throw`/`exit` for a JS-level exception** — always `#(Error(v), st')`. State mutations before the throw are preserved in `st'`. Panic only on internal invariant violation (unreachable slot kind, etc.). This is the M17 exception-ABI requirement: state must survive throws, so it rides the return tuple, not the BEAM exception mechanism.

## §5 `init_realm` + `t_globals` (`rt_js_builtins.gleam`)

```gleam
import twocore/runtime/rt_js_store.{type JsStore}
import twocore/runtime/rt_js_builtins/common.{type Realm, type BuiltinPair, type Handle}

/// Allocate every intrinsic into a FRESH store. The ONLY place intrinsics
/// are allocated (invariant §6). Returns the populated store and the Realm
/// handle table. Does NOT allocate globalThis (t_globals does that,
/// because globalThis needs the completed Realm).
pub fn init_realm(store0: JsStore) -> #(JsStore, Realm)

/// Allocate globalThis with every §19.1-§19.3 binding installed as a
/// property, plus the self-referencing "globalThis" property. Returns the
/// updated store and the Realm with `global_this` filled in.
pub fn t_globals(store: JsStore, realm: Realm) -> #(JsStore, Realm)

/// The single native dispatcher. Called by M4's t_call and by M8's job runner.
pub fn dispatch_native(st: St, tok: NativeToken, this: JsVal, args: List(JsVal), nt: Option(JsVal)) -> #(JsResult, St)
```

**`init_realm` allocation order** (each step: `let #(s, x) = <mod>.init(s, ...)`) — mirrors arc/builtins.gleam:54-456 exactly, because prototype-chain wiring depends on order:
```
 1. object_proto   = alloc_proto(s, prototype: None, props: {})           // root of all chains
 2. #(function, throw_type_error) = function.init(s, object_proto)        // Function.prototype→Object.prototype
 3. object         = object.init(s, object_proto, function.prototype)      // fills object_proto's methods
 4. array          = array.init(s, object_proto, function.prototype)
 5. errors         = error.init(s, object_proto, function.prototype)       // Error + 8 subclasses
 6. math           = math.init(s, object_proto, function.prototype)
 7. string         = string.init(s, object_proto, function.prototype)
 8. #(number, parse_int, parse_float, is_nan, is_finite) = number.init(...)
 9. boolean, regexp, date, promise
10. iterator_proto = alloc {[@@iterator]: ReturnThis}; proto→object_proto
11. #(iterator, iterator_helper_proto, wrap_for_valid) = iterator.init(s, iterator_proto, function.prototype)
12. array_iterator_proto, set_iterator_proto, map_iterator_proto, string_iterator_proto, regexp_string_iterator_proto
    = each: alloc {next: <Mod>IteratorNext, [@@toStringTag]: "<Tag>"}; proto→iterator_proto
13. generator      = generator.init(s, iterator_proto, function.prototype, function.constructor)
14. async_iterator_proto = alloc {[@@asyncIterator]: ReturnThis}; proto→object_proto
15. async_from_sync_iterator_proto = alloc {next,return,throw}; proto→async_iterator_proto
16. async_generator = generator.init_async(s, async_iterator_proto, function.prototype, function.constructor)
17. async_function_proto = init_async_function(s, function.prototype, function.constructor)
18. symbol, console, json, reflect, proxy, map, set, weak_map, weak_set, finalization_registry
19. disposable_stack, async_disposable_stack
20. array_buffer, shared_array_buffer, data_view, typed_array + 11 concrete
21. bigint, atomics
22. intl, temporal
23. eval, decodeURI, encodeURI, decodeURIComponent, encodeURIComponent, escape, unescape (7 global fns)
→ return #(s, Realm(..., global_this: rt_js_store.null_handle))   // placeholder
```

**`t_globals`** (mirrors arc/builtins.gleam:489-635):
- alloc globalThis object, prototype→object.prototype
- install 3 immutable props: `NaN`, `Infinity`, `undefined` (writable:F, enumerable:F, configurable:F)
- install ~60 builtin props (writable:T, enumerable:F, configurable:T): every constructor, every namespace, every global fn, plus 11 typed-array ctors (list at arc/builtins.gleam:533-606)
- install `globalThis` self-ref
- return `#(s, Realm(..realm, global_this: h))`

## §7 GC-trace hook for NativeToken (interface with M2)

M2's mark phase (port of arc/heap.gleam:486-537 + arc/gc_trace.gleam) walks slots. When it hits `KNative(tok, _)` or `KBound(...)`, it must enumerate the `Handle`s captured in the token. M6 provides:
```gleam
// rt_js_builtins/common.gleam
pub fn native_token_refs(tok: NativeToken) -> List(Handle)
```
Exhaustive `case` over every variant; returns `[]` for leaf variants, `[proto]` for `ErrorConstructor(proto)`, `[remaining, values, called, resolve]` for `PromiseAllResolveElement(...)`, etc. This is the direct port of arc/gc_trace.gleam's `native_fn_refs` (grep `gc_trace.gleam` for the arc precedent).

## §10 Invariants

1. **Single allocation site**: `init_realm` is the ONLY function that allocates intrinsic prototypes/constructors. No builtin impl allocates a prototype. Enforced by: intrinsics are only reachable via `st.realm.<field>`; there is no `alloc_proto` call outside `init` fns, and `init` fns are called only from `init_realm`.
2. **Prototype-chain termination**: every `Slot.prototype` chain reaches `realm.object.prototype` whose own `prototype` is `None`, in ≤5 hops. `init_realm` establishes this by construction (allocation order §5).
3. **Realm immutability**: `Realm` fields are never reassigned after `t_globals` returns. The objects they point to ARE mutable (user can `Object.prototype.foo = ...`), but the handles are stable.
4. **Rooting**: every Handle in `Realm` is a GC root. M2's root-set fn returns `realm_handles(st.realm) : List(Handle)` — a flat enumeration of every field (M6 provides `pub fn realm_handles(r: Realm) -> List(Handle)` in common.gleam).
5. **State preservation on throw**: every `dispatch` arm returns the up-to-date `St` even on `Error(_)`.
6. **NewTarget honoring**: every constructor variant checks `nt`; `Symbol/BigInt` throw on `Some`, `Map/Set/WeakMap/WeakSet/Promise/Proxy/FinalizationRegistry/DisposableStack/DataView/all-TypedArrays/ArrayBuffer/SharedArrayBuffer` throw on `None`.

## §11 Dependencies & build order

- **M6 depends on**: M1 (JsStore, Handle, JsSlot, ObjKind, Property, HostHooks), M3 (JsVal helpers/coercion — used inside impls), M4 (t_get_prop/t_set_prop/t_new_object — used inside impls that build result objects), rt_js_state (St type). M8 for Promise's microtask enqueue (`enqueue_job(st, Job) -> St`).
- **M6 is depended on by**: M4's `t_call` (via `dispatch_native`), M2 (via `native_token_refs` + `realm_handles`), M9 (resolve_js registers `"call"`/`"construct"`/`"realm_new"` ops), M14/M16 (emit `CallHost("js","make_function",...)` and constructor tokens), M19 (module init emits `init_realm` + `t_globals`).
- **Circular break**: `rt_js_builtins/common.gleam` (types only) ← rt_js_state ← rt_js_obj ← `rt_js_builtins/<mod>.gleam` ← `rt_js_builtins.gleam`. M4's `t_call` importing `rt_js_builtins.dispatch_native` would create obj→builtins→obj cycle; break it by having `rt_js_builtins.gleam` re-export `t_call` (i.e., `t_call` LIVES in rt_js_builtins.gleam, using rt_js_obj primitives, and M4 exports only the primitives). Alternatively pass `dispatch_native` as a callback — but re-homing `t_call` is simpler.

**REVISED PLACEMENT**: `t_call/t_construct` live in `rt_js_builtins.gleam` (not M4). M4 rt_js_obj exports only prototype-walk primitives (`t_get_prop`, `t_set_prop`, `t_define_prop`, `t_own_keys`, `t_get_proto`, `t_new_object`). This mirrors arc where `call.gleam` sits above both `builtins/*` and `ops/object`.
